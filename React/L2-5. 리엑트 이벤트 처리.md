# 리엑트 이벤트 처리

- 리엑트의 이벤트는 사용자의 인터렉션(클릭, 입력, 마우스 이동 등) 발생 시 EventHandler로 전달됨.
- React는 이벤트 그대로를 사용하지 않고, ** 합성 이벤트 ** 객체를 생성하여 핸들러로 저장됨.

## HTML이벤트 처리와 차이점

1. 이벤트 핸들러명: 카멜케이스로 작성(onclick X) -> (onClick O)
2. 이벤트 핸들러로 함수를 전달해 이벤트를 처리함
3. 모든 브라우저가 일관된 이벤트 처리를 위해 이벤트 객체 SyntheticEvent를 제공

- 이 객체(SyntheticEvent)를 통해 페이지 이동같은 기본동작 방지 가능
- 이벤트 발생한 DOM 요소 참조 가능

## 이벤트 핸들러 onClick에 함수 전달하기

- 예시1: props 이용, FrontComp)

```jsx
// 자식 컴포넌트
function ForntComp(props) {
  return (
    <>
      <li>
        <a
          href="/"
          onClick={() => {
            props.onMyEvent1();
          }}
        >
          프론트엔드
        </a>
      </li>
      <ul>
        <li>HTML5</li>
        <li>CSS3</li>
        <li>JavaScript</li>
      </ul>
    </>
  );
}
// 부모 컴포넌트
function app() {
  return (
    <>
      <h2>React-Event</h2>
      <ol>
        <ForntComp
          onMyEvent1={() => {
            alert("프론트엔드 클릭됨(부모전달)");
          }}
        />
      </ol>
    </>
  );
}
export default app;
```

- 예시2: props이용, BackComp)

```jsx
// 자식 컴포넌트
function FrontComp(props) {
  const BackComp = ({ onMyEvent }) => {
    return (
      <>
        <li>
          <a
            href="/"
            onClick={(event) => {
              event.preventDefault();
              onMyEvent("백안드 클릭됨(자식전달");
            }}
          >
            백엔드
          </a>
        </li>
        <ul>
          <li>JAVA</li>
          <li>Oracle</li>
          <li>MySQL</li>
        </ul>
      </>
    );
  };
}
// 부모 컴포넌트
function App() {
  return (
    <>
      <h2>React-Event</h2>
      <BackComp
        onMyEvent={(msg) => {
          alert(msg);
        }}
      />
    </>
  );
}
```

### FrontComp, BackComp의 차이

- 탑 - 다운 방식: 부모에서 자식으로 이동함(예. props)
- 탑-다운 방식에서는 자식이 부모에게 컴포넌트 데이터를 넘길 수 없음
- \*\*\* 그러나 이벤트를 통해 자식컴포넌트를 부모 컴포넌트로 이동시킬 수 있음

### Back-Top 방식

- 예시의 Back-Top방식의 컴포넌트는 부모컴포넌트의 onClick 이벤트로 onMyEvent가 호출되면서
  부모 자식컴포넌트로 올라간다.

### preventDefault()

- onClick 이벤트 발생 시 화면깜빡임(replace)가 발생되는 문제가 발생
- 깜빡이기 이전에 event.preventDefault()를 실행하면, 이벤트 처리의 동작을 차단하게 됨(깜빡임 이슈 해결)
